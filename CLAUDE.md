# プロジェクトガイドライン

このプロジェクトはプロジェクトセカイの楽曲をCRUD APIを用いて管理する管理Webページになります

## 基本ルール

### 使用言語

ユーザーとの会話は日本語で行ってください。

## ドキュメント記述規約

### 技術用語の扱い

技術用語を不用意に日本語変換しないこと。

* 技術用語は原則として英語のまま使用すること
* 日本語訳が分かりにくい場合は、英語表記を優先すること

### 図の記述

* 図を記述する場合は、mermaidを使用すること

### 句読点の使用

日本語文章には`.`（ピリオド）ではなく`。`（句点）を使うこと。

* 日本語の文末には必ず`。`を使用すること
* コード中のコメントに`。`や`.`は不要
* 英語の文章では`.`を使用すること

## 技術スタック

### フロントエンド

| カテゴリ | ツール | 備考 |
|---------|--------|------|
| フレームワーク | Vue 3 | ページ数が少ないため軽量なVueを採用 |
| 言語 | TypeScript | 型安全性を確保 |
| ビルドツール | Vite | 高速なビルド・開発サーバー |
| パッケージマネージャー | pnpm | 高速で効率的なパッケージ管理 |

### 品質管理・テスト

| カテゴリ | ツール | 理由 |
|---------|--------|------|
| 品質管理 | Biome | Linting・Formattingを統一 |
| ユニットテスト | Vitest | Vue + TypeScript環境に最適。高速で軽量 |
| テストユーティリティ | Vue Test Utils | Vue公式のテストユーティリティ |
| E2Eテスト | Playwright | クロスブラウザ対応で管理画面のテストに適切 |

### その他の推奨ツール

| ツール | 用途 |
|--------|------|
| fetch API | APIとの通信 |
| Tailwind CSS | UI構築（軽量で迅速） |

### テストツール選定の理由

**Vitest + Vue Test Utils + Playwright** の構成としています。

* **Vitest**：Viteベースで開発環境と統一でき、高速なテスト実行が可能。Vue 3との相性が良い。
* **Vue Test Utils**：Vue公式のテストライブラリで、コンポーネントテストに特化。
* **Playwright**：管理画面のE2Eテストに必要。複数ブラウザでの動作確認も可能。

ページ数が少ないため、ユニットテスト中心でシンプルに保ちながら、重要なフローはE2Eテストでカバーする想定です。
